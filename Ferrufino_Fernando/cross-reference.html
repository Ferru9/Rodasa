    • 3.2 Minimum Requirement Features

    3.2.1 HTML5

    • 3.2.1 A Appropriate HTML5 section elements tags, including use of HTML5 structural tags used in sectioning (ex: the body, header, section and footer tag on the index html.)
    • 3.2.1 B.I Inline images static (ex: the image on the Our Specialty section of the about page.)
    • 3.2.1 B.II Inline images clickable (ex: the image on the History section of the about page.)
    • 3.2.1 C Background images (ex: the background image that covers all the header on the product page.)
    • 3.2.1 D Use of appropriate special characters. (ex: the use of the copyright symbol on the bottom of the page )
    • 3.2.1 E Use of appropriate tags. (ex: using the right encoding UTF-8, and the right view port)
    • 3.2.1 F A footer on the home page with appropriate links, other than the links provided in the top navigation. (ex: there is a footer in every page on the website, it contains the location of the company and links to all the products and services. Also the link to all the companies social media.)
    • 3.2.1 G The footer should also include when website was last updated. This must be done dynamically using JavaScript DOM.

    3.2.2 CSS3

    • 3.2.2 A.I Use CSS3 to format the appearance of paragraphs (ex: styling the margin and the line height of each paragraph.)
    • 3.2.2 A.II Use CSS3 to format the appearance of lists (ex: styling the list on the footer, changing the color and removing the style, and adding some hover effects. )
    • 3.2.2 A.III Use CSS3 to format the appearance of header (ex: styling the header, by adding a background images and appropriate positioning. )
    • 3.2.2 A.IV Use CSS3 to format the appearance of background images (ex: placing a linear gradient to the background image of the header.)
    • 3.2.2 A.V Use CSS3 to format the appearance of table (ex: changing the color and adding borders and layout of the table)
    • 3.2.2 A.VI Use CSS3 to format the appearance of the form (ex: styling the labels, buttons legends of the form.)
    • 3.2.2 B.I Use CSS3 contextual selectors (e1, e2, e3) (ex: when selecting the body, h1, h2, h3, p to set the margin to 0.)
    • 3.2.2 B.II Use CSS3 contextual selectors (e f) (ex: when selecting the header nav ul to display none for the mobile version of the header.)
    • 3.2.2 B.III Use CSS3 contextual selectors (e > f) (ex: when selecting .split>* meaning inside the class split select everything and make its flex basis 100%)
    • 3.2.2 B.IV Use CSS3 contextual selectors (e + f) (ex: when using input[type="radio"]:checked+label so this will select the input of type radio with the label and change its color.)
    • 3.2.2 B.V Use CSS3 2 attribute selectors (ex: when using the attribute selector input[type="radio"] which will target the input with type radio. And also when using the input[type='submit'] which will target the input of type submit)
    • 3.2.2 C Use of CSS3 styles for color (RGBA & HSLA) (ex: changing the color on the nav, hsl is used. Also the green color from the logo is rgba)
    • 3.2.2 D Use of CSS3 code for progressive enhancement (ex: the font, 'raleway' font is used along all the webpage, however sans serif is used in case a browser doesn't support it. Following progressive enhancement)
    • 3.2.2 E.I Use of dynamic pseudo-classes to highlight hyperlinks states of: link (ex: change the decoration of the link when the link has not been touched in any way on the footer)
    • 3.2.2 E.II Use of dynamic pseudo-classes to highlight hyperlinks states of: visited (ex: change the color when the link is visited on the footer )
    • 3.2.2 E.III Use of dynamic pseudo-classes to highlight hyperlinks states of: active (ex: make the link underlined when is active on the footer )
    • 3.2.2 E.IV Use of dynamic pseudo-classes to highlight hyperlinks states of: hover (ex: make the link underlined when the mouse hovers on it, on the footer )
    • 3.2.2 E.V Use of dynamic pseudo-classes to highlight hyperlinks states of: focus (ex: make the link underlined when the link is focused, on the footer)
    • 3.2.2 E.VI two structural pseudo-classes: first-of-type and last-of-type (ex: creating the footer with a column system, making the column have no padding on the right and on the left depending on what column is.)
    • 3.2.2 F.I Use of CSS3 background styles (background size) (ex: making the background image cover on the header section)
    • 3.2.2 F.II Use of CSS3 background styles (background-clip) (ex: making the background image clip on border-box so it covers all the parts of the screen.
    • 3.2.2 F.III Use of CSS3 background styles (background-origin) (ex: on the about section making the background image origin border-box)
    • 3.2.2 F.IV Use of CSS3 padding, margins and borders (ex: setting the margin at 0 on, the body, h1,h2,h3,p also setting the padding to 4rem on the section, and the border to collapse on the table.)
    • 3.2.2 F.V Use of an appropriate border style (ex: setting the border styles on the buttons)
    • 3.2.2 F.VI Use of rounded corners (ex: making the buttons on the reset and submit of the form with rounded corners)
    • 3.2.2 G Use of a drop cap created with CSS3 on your home page (ex: making the R on Rodasa as a drop cap on the about us box on the home page.)
    • 3.2.2 H.I Use each of the following page layout techniques: Grid (ex: the product and the service body is composed of a grid layout.)
    • 3.2.2 H.II Use each of the following page layout techniques: Flex (ex: the about page is composed of a flex layout for the body.)
    • 3.2.2 H.III Use each of the following page layout techniques: Float (ex: the section of the products of the home page uses float left so images stay together floated of their left.)
    • 3.2.2 H.IV Use each of the following page layout techniques: Positioning (ex: the use of relative and absolute positioning on the Who we are section of the home page)
    • 3.2.2 I Use of CSS3 text and box shadows (ex: the uso of text-shadow on the header title and a box shadow for the products of the page)
    • 3.2.2 J Use of CSS3 transform rotate (ex: rotate the two images that are translucent)
    • 3.2.2 K Use of CSS3 opacity features to two images to appear semi-transparent (ex: setting 2 images on the about page with opacity of 0.5 that are semi-transparent)
